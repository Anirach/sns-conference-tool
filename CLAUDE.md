# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Status

**Pass 1 (current):** Fully navigable Next.js frontend driven entirely by MSW mock data. Flutter mobile shell with bridge stubs. No real backend yet.

**Pass 2 (planned):** Spring Boot 3.3 backend replaces MSW mocks. Real native mobile services (GPS, QR, FCM, OAuth).

## Commands

### Web (`cd web`)
```bash
pnpm install       # install deps
pnpm dev           # dev server → http://localhost:3000
pnpm build         # production build
pnpm lint          # ESLint (next/core-web-vitals)
pnpm typecheck     # tsc --noEmit
```

### Mobile (`cd mobile`)
```bash
flutter pub get    # install deps
flutter run        # run on device/emulator
flutter analyze    # Dart static analysis
```
Android emulator loads `http://10.0.2.2:3000`; iOS simulator loads `http://localhost:3000`.
Override with `--dart-define=FRONTEND_ORIGIN=https://staging.example.com`.

### Infrastructure (`cd infra`)
```bash
docker compose -f docker-compose.dev.yml up -d
# starts: PostgreSQL+PostGIS :5432, Redis :6379, MinIO :9000/:9001, MailHog :1025/:8025
```

## Architecture

### Three-layer structure
```
web/      → Next.js 14 (App Router) — primary UI
mobile/   → Flutter 3.22 WebView shell + native bridge
backend/  → Spring Boot 3.3 (Pass 2 only, currently empty)
```

### JS ↔ Flutter Bridge
The most non-obvious system in this repo. Every cross-boundary call uses `{ id, type, payload }` JSON messages.

- **Web → Native**: `window.FlutterBridge.postMessage(json)` — for GPS, QR scan, file picker, secure storage, SNS OAuth, push tokens
- **Native → Web**: Flutter evaluates JS callback — for push notifications, connectivity changes, app resume
- Web side: `web/lib/bridge/` (client.ts, types.ts, mock.ts for browser fallback)
- Flutter side: `mobile/lib/bridge/` (js_bridge.dart dispatches to native services in `mobile/lib/native/`)

### Mock API layer (Pass 1)
`NEXT_PUBLIC_MOCK_API=1` activates MSW. All `/api/*` requests are intercepted by `web/lib/api/mocks/handlers.ts` (500+ lines). The real API client in `web/lib/api/` mirrors the same contract — switching to real backend is env-variable-only.

### State management (web)
- **Zustand** (`web/lib/state/`): auth (JWT, user ID), current event, chat messages
- **TanStack React Query**: server state with caching/refetching
- **Axios** (`web/lib/api/axios.ts`): JWT injection + silent token refresh interceptor

### Real-time (designed, not yet active)
STOMP over WebSocket. Subscriptions planned for:
- `/user/queue/matches` — proximity match updates via Redis Pub/Sub
- `/user/queue/chat/{otherId}` — peer chat
- `/topic/event/{eventId}/status` — event announcements

STOMP client setup lives in `web/lib/ws/`; mock fallback in `web/lib/ws/mock.ts`.

### Flutter local persistence
Isar database (`mobile/lib/storage/`) caches match entities and settings on-device. Secure storage holds JWT + refresh tokens via `flutter_secure_storage`.

## Key conventions

- Path alias `@/*` maps to `web/` root (configured in tsconfig.json)
- Tailwind custom colors: `brand-*` and `brass-*` palettes; custom keyframes `laser-scan`, `pulse-halo`, `typing-bounce`
- All fixture/mock data lives in `web/lib/fixtures/` — types defined in `fixtures/types.ts`
- `web/public/mockServiceWorker.js` is auto-generated by MSW — do not edit manually
- shadcn/ui components are in `web/components/ui/` and are owned (not node_modules)
