# backend/

Spring Boot 3.3 / Java 21 implementation per [docs/SNS-system.md §6](../docs/SNS-system.md).

## Status

- **Phase 1**: Auth + Profile + GDPR export (profile only). Flyway V1–V2. Real RS256 JWT + rotating refresh tokens.
- **Phase 2**: Events + Interests + Matching with PostGIS vicinity queries, TF-IDF similarity, async recompute.
- **Phase 3**: Real-time chat (WebSocket + STOMP with JWT CONNECT auth), push outbox with at-least-once delivery, device token registration, match-found + chat events driving notifications.
- **Phase 4**: SNS OAuth2 link/unlink (Facebook, LinkedIn) with AES-256-GCM token encryption, GDPR export aggregator + soft/hard-delete cron, CSP + HSTS security headers.
- **Phase 5**: Observability (JSON logs, `X-Request-Id` correlation, Micrometer + OTel), Prometheus alert rules, Grafana dashboard, Helm chart with HPA + PDB + CronJob, on-call runbooks.

## First-time setup

The Gradle wrapper jar is not checked in yet (generated by `gradle wrapper`). Bootstrap once:

```bash
# Requires Gradle 8.10+ and Java 21 on PATH
gradle wrapper --gradle-version 8.10
```

## Build & run

```bash
# From repo root: start dependencies (Postgres + MailHog + Redis + MinIO)
cd infra && docker compose -f docker-compose.dev.yml up -d

# Build and run locally
cd backend
./gradlew :app:bootRun
```

Endpoints:
- `http://localhost:8080/actuator/health` — liveness
- `http://localhost:8080/.well-known/jwks.json` — public key for access-token verification
- `http://localhost:8080/swagger-ui.html` — Springdoc UI
- `http://localhost:8080/api/auth/**` — public
- `http://localhost:8080/api/profile`, `/api/users/me/export` — JWT-gated

## End-to-end smoke (curl)

```bash
# 1. Register (returns 202)
curl -sS -X POST localhost:8080/api/auth/register \
  -H 'Content-Type: application/json' \
  -d '{"email":"ada@example.com"}'

# 2. Grab the TAN — in dev-mode it is always "123456"; in prod check MailHog at http://localhost:8025

# 3. Verify (returns verificationToken)
TOKEN=$(curl -sS -X POST localhost:8080/api/auth/verify \
  -H 'Content-Type: application/json' \
  -d '{"email":"ada@example.com","tan":"123456"}' | jq -r .verificationToken)

# 4. Complete (returns access+refresh)
TOKENS=$(curl -sS -X POST localhost:8080/api/auth/complete \
  -H 'Content-Type: application/json' \
  -d "{\"verificationToken\":\"$TOKEN\",\"firstName\":\"Ada\",\"lastName\":\"Lovelace\",\"password\":\"analytical-engine\"}")
ACCESS=$(echo "$TOKENS" | jq -r .accessToken)

# 5. Fetch profile
curl -sS localhost:8080/api/profile -H "Authorization: Bearer $ACCESS"
```

## Tests

```bash
./gradlew :app:test               # unit/slice tests
./gradlew :app:integrationTest    # Testcontainers-backed flows (requires Docker)
```

The integration test spins up a `postgis/postgis:15-3.4` container via Spring Boot's `@ServiceConnection` and walks the full registration flow end-to-end.

## Running with docker-compose

```bash
cd infra
docker compose -f docker-compose.dev.yml --profile backend up --build
```

This builds the backend image from `backend/app/Dockerfile` and wires it to Postgres + MailHog.

## JWT keys

- Dev: if `JWT_PRIVATE_KEY` / `JWT_PUBLIC_KEY` are unset, an ephemeral RS256 keypair is generated at boot (tokens invalidate on restart).
- Prod: supply PEM-encoded PKCS#8 private + X.509 public keys via env. Rotate by deploying new keys and draining old tokens via the refresh-token revocation path.

## Wiring the web frontend to the real backend

In `web/.env.local`:

```
NEXT_PUBLIC_MOCK_API=1,-auth,-profile    # mocks ON for everything except auth/profile
BACKEND_PROXY_TARGET=http://localhost:8080
```

Next's dev server rewrites `/api/*` and `/.well-known/*` to the backend. Change the mock CSV as each phase ships.
